## <a name="remarks"></a>주의  
 파생 된 `CWnd`, Windows의 창 개체의 모든 기능 및 특정 기능을 추가 OLE를 이벤트 발생 및 기능과 같은 메서드 및 속성을 지원 하기 위해이 클래스를 상속 합니다.  
  
 OLE 컨트롤 OLE 컨테이너 응용 프로그램에 삽입할 수와 이벤트 발생 및 노출을 메서드 및 속성을 컨테이너의 양방향 시스템을 사용 하 여 컨테이너와 통신 합니다. 표준 OLE 컨테이너에만 OLE 컨트롤의 기본 기능 지원 참고 합니다. OLE 컨트롤의 확장된 기능을 지원할 수지 않습니다. 이벤트 발생 이벤트가 컨트롤에서 발생 하는 특정 작업의 결과로 컨테이너에 전송 되 면 발생 합니다. 차례로 컨테이너 컨트롤에서 노출 되는 일련의 메서드 및 속성의 멤버 함수와 유사와 c + + 클래스의 데이터 멤버를 사용 하 여 통신 합니다. 따라서 개발자를 컨트롤의 모양을 제어 하 고 특정 작업이 발생 하면 컨테이너에 알리는을 수 있습니다.  
  
## <a name="windowless-controls"></a>창 없는 컨트롤  
 OLE 컨트롤 창 없이 사용 되는 내부 활성화 될 수 있습니다. 창 없는 컨트롤 몇 가지 중요 한 이점이 있습니다.  
  
-   투명 한 사각형이 아닌 창 없는 컨트롤 수 있습니다.  
  
-   창 없는 컨트롤 개체의 인스턴스 크기와 만든 시간을 줄이기  
  
 컨트롤에 창이 필요 하지 않습니다. 서비스 창에서 제공 하는 단일 공유 창 (일반적으로 컨테이너의) 하 고 약간의 디스패치 코드를 통해 쉽게 제공할 수 있습니다. 창 개체에는 불필요 한 복잡성 대부분은.  
  
 창 없는 활성화를 사용 하는 경우 (창이 필요)는 컨테이너는 컨트롤의 창으로 제공은 서비스를 제공 합니다. 예를 들어 컨트롤이 키보드 포커스를 쿼리하거나 마우스 캡처를 쿼리 하 장치 컨텍스트를 가져와야 하는 경우 이러한 작업이 컨테이너에 의해 관리 됩니다. `COleControl` [작업 창 없는 멤버 함수](http://msdn.microsoft.com/en-us/e9e28f79-9a70-4ae4-a5aa-b3e92f1904df) 컨테이너에서 이러한 작업을 호출 합니다.  
  
 컨테이너 대리자 입력 된 메시지를 컨트롤의 창 없는 활성화를 사용 하는 경우 `IOleInPlaceObjectWindowless` 인터페이스 (의 확장인 [IOleInPlaceObject](http://msdn.microsoft.com/library/windows/desktop/ms692646) 창 없는 지원에 대 한). `COleControl`좌표를 적절히 마우스를 조정한 후이 인터페이스의 구현을 컨트롤의 메시지 맵을 통해 이러한 메시지를 디스패치합니다. 해당 항목을 메시지 맵에 추가 하 여 일반적인 창 메시지와 마찬가지로 이러한 메시지를 처리할 수 있습니다.  
  
 창 없는 컨트롤에서 항상 사용 해야는 `COleControl` 멤버 함수는 해당 하는 대신 `CWnd` 멤버 함수 또는 관련된 Windows API 함수입니다.  
  
 경우에 활성화, 될 하지만 활성 비활성 전환 하는 데 필요한 작업 양을 배로 전환의 속도 작동이 OLE 컨트롤 개체 창을 만들 수도 있습니다. 이것이 문제가 되는 경우가 있습니다: 예를 들어 입력란의 표를 고려 합니다. 때 열을 위쪽 및 아래쪽 커서를 각 컨트롤 이어야 합니다 내부 활성화 및 비활성화 합니다. 활성/비활성 전환의 속도를 스크롤 속도를 직접 영향을 줍니다.  
  
 OLE 제어 프레임 워크는 개발 방법에 대 한 자세한 내용은 문서를 참조 [MFC ActiveX 컨트롤](../../mfc/mfc-activex-controls.md) 및 [개요: MFC ActiveX 컨트롤 프로그램 만들기](../../mfc/reference/mfc-activex-control-wizard.md)합니다. OLE 컨트롤, 창 없는 및 깜박임 컨트롤을 포함 하 여 최적화에 대 한 내용은 [MFC ActiveX 컨트롤: 최적화](../../mfc/mfc-activex-controls-optimization.md)합니다.

##  <a name="getactivationpolicy"></a>COleControl::GetActivationPolicy  
 지 원하는 컨트롤의 기본 활성화 동작을 변경 하는 `IPointerInactive` 인터페이스입니다.  
  
```  
virtual DWORD GetActivationPolicy();
```  
  
### <a name="return-value"></a>반환 값  
 플래그의 조합 된 **POINTERINACTIVE** 열거 합니다. 가능한 플래그는:  
  
 **POINTERINACTIVE_ACTIVATEONENTRY**  
 개체는 내부 마우스가 마우스 이동 작업 중 들어올 때 활성화 해야 합니다.  
  
 **POINTERINACTIVE_DEACTIVATEONLEAVE**  
 마우스가 마우스 이동 작업 중 개체를 벗어날 때 개체를 비활성화 해야 합니다.  
  
 **POINTERINACTIVE_ACTIVATEONDRAG**  
 개체 위로 끄는 동안 마우스를 끌 때 활성화 될와 작업을 삭제 합니다.  
  
### <a name="remarks"></a>주의  
 경우는 `IPointerInactive` 인터페이스를 사용 하면 컨테이너를 위임 하 게 `WM_SETCURSOR` 및 `WM_MOUSEMOVE` 메시지를 합니다. `COleControl`좌표를 적절히 마우스를 조정한 후이 인터페이스의 구현을 컨트롤의 메시지 맵을 통해 이러한 메시지를 디스패치합니다.  
  
 컨테이너가 받는 때마다는 `WM_SETCURSOR` 또는 `WM_MOUSEMOVE` 메시지는 비활성 개체를 지 원하는 위에 마우스 포인터를 `IPointerInactive`를 호출 해야 `GetActivationPolicy` 반환 플래그와 인터페이스에는 **POINTERINACTIVE** 열거형.  
  
 해당 항목을 메시지 맵에 추가 하 여 일반적인 창 메시지와 마찬가지로 이러한 메시지를 처리할 수 있습니다. 처리기를 사용 하지 않도록는 `m_hWnd` 멤버 변수 (또는를 사용 하는 모든 멤버 함수) 먼저 그 값이 아닌 검사 **NULL**합니다.  
  
 수행 이상 마우스 커서를 설정 및/또는 마우스 이동 이벤트를 발생 시키는 위한 모든 개체를 같은 특수 한 시각적 피드백을 제공 하는 대로 반환 해야는 **POINTERINACTIVE_ACTIVATEONENTRY** 플래그를 지정 하 고 활성 상태인 경우에 사용자 의견을 그립니다. 컨테이너 그 위치에서 즉시 정품 인증 하 고 동일한 메시지에 대 한 호출을 트리거한 전달 해야이 플래그를 반환 하는 개체 `GetActivationPolicy`합니다.  
  
 모두는 **POINTERINACTIVE_ACTIVATEONENTRY** 및 **POINTERINACTIVE_DEACTIVATEONLEAVE** 플래그 반환 되는 다음 개체가 마우스가 개체 위에 있을 때만 활성화 됩니다. 만 **POINTERINACTIVE_ACTIVATEONENTRY** 플래그 반환 되 면 다음 개체가 때 마우스 개체를 처음 실행 한 후에 활성화 됩니다.

##  <a name="getcontrolflags"></a>COleControl::GetControlFlags  
 제어 플래그 설정을 검색합니다.  
  
```  
virtual DWORD GetControlFlags();
```  
  
### <a name="return-value"></a>반환 값  
 ControlFlags 열거형에서 플래그의 조합 된 ORed입니다.  
  
 `enum ControlFlags {`  
  
 `fastBeginPaint = 0x0001,`  
  
 `clipPaintDC = 0x0002,`  
  
 `pointerInactive = 0x0004,`  
  
 `noFlickerActivate = 0x0008,`  
  
 `windowlessActivate = 0x0010,`  
  
 `canOptimizeDraw = 0x0020,`  
  
 `};`  
  
### <a name="remarks"></a>주의  
 기본적으로 `GetControlFlags` 반환 `fastBeginPaint | clipPaintDC`합니다.  
  
 `fastBeginPaint`  
 경우 설정, 사용 하 여 begin-페인트 함수에 맞게 조정할 수 대신 OLE 컨트롤의 [BeginPaint](http://msdn.microsoft.com/library/windows/desktop/dd183362) API (기본적으로 설정 됨).  
  
 `clipPaintDC`  
 설정 되지 않은 경우에 대 한 호출을 사용 하지 않도록 설정, `IntersectClipRect` 수행한 `COleControl` 및 작은 속도가 빨라집니다. 창 없는 활성화를 사용 하는 경우 플래그는 영향을 주지 않습니다.  
  
 `pointerInactive`  
 경우 설정, 사용 하 여 사용자 컨트롤이 활성화 되지 않은 동안 마우스 상호 작용을 제공 `COleControl`의 구현의 `IPointerInactive` 인터페이스를 기본적으로 비활성화 됩니다.  
  
 `noFlickerActivate`  
 경우 설정, 추가 그리기 작업과 깜빡임을 제거 합니다. 컨트롤의 비활성 및 활성 상태에서 자신을 동일 하 게 그릴 때 사용 됩니다. 창 없는 활성화를 사용 하는 경우 플래그는 영향을 주지 않습니다.  
  
 `windowlessActivate`  
 경우 설정, 창 없는 활성화를 사용 하는 컨트롤을 나타냅니다.  
  
 `canOptimizeDraw`  
 경우 설정, 컨테이너에서 지 원하는 경우 컨트롤 최적화 된 그리기를 수행 함을 나타냅니다.  
  
 에 대 한 자세한 내용은 `GetControlFlags` OLE 컨트롤의 다른 최적화 참조 및 [ActiveX 컨트롤: 최적화](../../mfc/mfc-activex-controls-optimization.md)합니다.

##  <a name="getdc"></a>COleControl::GetDC  
 창 없는 개체에 대 한 get 화면 (또는 호환) 장치 컨텍스트를 해당 컨테이너에서를 제공합니다.  
  
```  
CDC* GetDC(
    LPCRECT lprcRect = NULL,
    DWORD dwFlags = OLEDC_PAINTBKGND);
```  
  
### <a name="parameters"></a>매개 변수  
 *lprcRect*  
 사각형 창 없는 컨트롤에 대 한 포인터를 컨트롤의 클라이언트 좌표에서 다시 그릴 하려고 합니다. **NULL** 전체 개체의 범위를 의미 합니다.  
  
 `dwFlags`  
 그리기 장치 컨텍스트의 특성입니다. 선택할 수 있습니다.  
  
- **OLEDC_NODRAW** 나타냅니다 개체가 모든 그리기 하지만 단순히는 표시 장치에 대 한 정보를 수행 하는 장치 컨텍스트를 사용 하지 않습니다. 단순히 컨테이너 추가적인 처리 없이 창의 DC를 전달 해야 합니다.  
  
- **OLEDC_PAINTBKGND** DC를 반환 하기 전에 컨테이너 배경을 그릴를 요청 합니다. 개체는 투명 한 배경으로 영역을 다시 그려야 하는 것에 대 한 연결 된 DC를 요청 하는 경우이 플래그를 사용 해야 합니다.  
  
- **OLEDC_OFFSCREEN** 개체가 다음 화면으로 복사 해야 하는 화면에서 벗어난 비트맵으로 렌더링 하려고 하는 컨테이너에 알립니다. 개체는 그리기 작업을 수행 하려고 하는 다양 한 깜박임을 생성할 때이 플래그를 사용 해야 합니다. 컨테이너는 또는 not이 요청을 처리할 수 있습니다. 그러나이 플래그를 설정 하지 않으면 컨테이너 해야 다시 전달할는 화면 DC입니다. 이렇게 하면 선택 영역을 표시 하는 등 직접 화면 작업을 수행 하는 개체 (통해는 **XOR** 작업).  
  
### <a name="return-value"></a>반환 값  
 컨테이너에 대 한 디스플레이 장치 컨텍스트에 대 한 포인터 `CWnd` 클라이언트 영역 경우 성공이 고; 그렇지 않으면, 반환 값은 **NULL**합니다. 디스플레이 장치 컨텍스트 이후의 GDI 함수에서 컨테이너의 창의 클라이언트 영역에서 그리는 데 사용할 수 있습니다.  
  
### <a name="remarks"></a>주의  
 [ReleaseDC](#releasedc) 컨텍스트 해제는 페인트 한 후에 멤버 함수를 호출 해야 합니다. 호출할 때 `GetDC`, 개체 자체 클라이언트 좌표로 그릴 하려는 사각형을 전달 합니다. `GetDC`이 컨테이너 클라이언트 영역의 좌표를 변환합니다. 개체는 원하는 그리기 사각형의 크기를 검색할 수는 자체 클라이언트 영역 사각형 보다 큰 요청 하지 [GetClientRect](#getclientrect)합니다. 이 위치에 의도 하지 않은 실수로 그리기에서 개체를 방지 합니다.

##  <a name="onqueryhitpoint"></a>COleControl::OnQueryHitPoint  
 컨테이너에 대 한 응답에서 프레임 워크에서 호출 **IViewObjectEx::QueryHitPoint** 요청 합니다.  
  
```  
virtual BOOL OnQueryHitPoint(
    DWORD dwAspect,
    LPCRECT pRectBounds,
    POINT ptlLoc,
    LONG lCloseHint,
    DWORD* pHitResult);
```  
  
### <a name="parameters"></a>매개 변수  
 `dwAspect`  
 개체는 표시 하는 방법을 지정 합니다. 유효한 값은 열거형에서 가져온 [DVASPECT](http://msdn.microsoft.com/library/windows/desktop/ms690318) 또는 **DVASPECT2**합니다.  
  
 `pRectBounds`  
 에 대 한 포인터는 `RECT` OLE 컨트롤 클라이언트 영역의 경계 사각형을 지정 하는 구조입니다.  
  
 `ptlLoc`  
 에 대 한 포인터는 **가리킨** 구조를 적중 검사할 지점을 지정 합니다. 지점은 OLE 클라이언트 영역 좌표로 지정 됩니다.  
  
 `lCloseHint`  
 "Close" 지점에 적중에 대 한 검사를 정의 하는 거리입니다.  
  
 `pHitResult`  
 적중 쿼리 결과에 대 한 포인터입니다. 다음 값 중 하나입니다.  
  
- **HITRESULT_OUTSIDE** `ptlLoc` OLE 외부 개체를 닫지 됩니다.  
  
- **HITRESULT_TRANSPARENT** *ptlLoc* 이 이미지 근접 하지 않지만 OLE 개체의 경계 내에 있습니다. 투명 한 원 중간 지점 수 예를 들어 **HITRESULT_TRANSPARENT**합니다.  
  
- **HITRESULT_CLOSE** `ptlLoc` 내부 또는 외부 OLE 개체가 있지만 가까이 개체 내부로 간주 합니다. 작은, 씬, 세부 개체는이 값을 사용할 수 있습니다. 점이 경계 밖에 있는 것 하는 경우에 있을 수 있습니다 개체의 사각형 닫기 수 (이 과정이 필요 작은 개체에 도달).  
  
- **HITRESULT_HIT** `ptlLoc` 내 개체의 이미지입니다.  
  
### <a name="return-value"></a>반환 값  
 적중된 결과; 성공적으로 반환 하는 경우 0이 아닌 그렇지 않으면 0입니다. 적중은 OLE 컨트롤의 표시 영역으로 겹쳐진입니다.  
  
### <a name="remarks"></a>주의  
 지정된 된 지점 개체의 표시 사각형에 겹치는 지 여부를 쿼리 (지점에 도달). `QueryHitPoint`사각형이 아닌 개체에 대 한 적중 테스트를 재정의할 수 있습니다.

##  <a name="onrenderdata"></a>COleControl::OnRenderData

  
### <a name="parameters"></a>매개 변수  
 `lpFormatEtc`

### <a name="return-value"></a>반환 값  
 성공하면 0이 아니고, 그렇지 않으면 0입니다.

 경우 *tymed-> lpStgMedium* 는 **TYMED_NULL**, **STGMEDIUM** 할당 되 고에 지정 된 대로 입력 해야 *tymed-> lpFormatEtc*합니다. 그렇지 않은 경우 **TYMED_NULL**, **STGMEDIUM** 데이터와 함께에서 채워야 합니다.

 자세한 내용은 참조는 **FORMATETC** 및 **STGMEDIUM** 구조체에 [!INCLUDE[winSDK](../../atl/includes/winsdk_md.md)]합니다.

##  <a name="onwindowlessmessage"></a>COleControl::OnWindowlessMessage  
 컨테이너에 대 한 응답에서 프레임 워크에서 호출 **IOleInPlaceObjectWindowless::OnWindowMessage** 요청 합니다.  
  
```  
virtual BOOL OnWindowlessMessage(
    UINT msg,
    WPARAM wParam,
    LPARAM lParam,
    LRESULT* plResult);
```  
  
### <a name="parameters"></a>매개 변수  
 `msg`  
 Windows에 의해 전달 된 메시지 식별자입니다.  
  
 `wParam`  
 Windows에서 성공 합니다. 메시지 관련 추가 정보를 지정합니다. 이 매개 변수의 내용을의 값에 따라 달라 집니다는 `msg` 매개 변수입니다.  
  
 `lParam`  
 Windows에서 성공 합니다. 메시지 관련 추가 정보를 지정합니다. 이 매개 변수의 내용을의 값에 따라 달라 집니다는 `msg` 매개 변수입니다.  
  
 *plResult*  
 Windows 결과 코드입니다. 메시지 처리의 결과 지정 하 고 전송 되는 메시지에 따라 달라 집니다.  
  
### <a name="return-value"></a>반환 값  
 성공하면 0이 아니고, 그렇지 않으면 0입니다.  
  
### <a name="remarks"></a>주의  
 창 없는 컨트롤에 대 한 창 메시지를 처리 합니다. `COleControl``OnWindowlessMessage` 메시지 마우스 및 키보드 메시지 이외의 창 메시지에 사용 해야 합니다. `COleControl`제공 [SetCapture](#setcapture) 및 [SetFocus](#setfocus) 특히 마우스 캡처 및 키보드 포커스에 대 한 가져오려는 창 없는 OLE 개체입니다.  
  
 창 없는 개체에는 창 필요가 없기 때문에 컨테이너 디스패치 메시지를 사용 하는 메커니즘이 필요 합니다. 창 없는 OLE 개체를 해당 컨테이너를 통해 메시지를 가져옵니다는 `OnWindowMessage` 메서드는 `IOleInPlaceObjectWindowless` 인터페이스 (의 확장인 [IOleInPlaceObject](http://msdn.microsoft.com/library/windows/desktop/ms692646) 창 없는 지원에 대 한). `OnWindowMessage`사용 하지 않는 한 `HWND` 매개 변수입니다.  
  
##  <a name="parenttoclient"></a>COleControl::ParentToClient  
 좌표 변환 `pPoint` 를 클라이언트 좌표로 합니다.  
  
```  
virtual UINT ParentToClient(
    LPCRECT lprcBounds,
    LPPOINT pPoint,
    BOOL bHitTest = FALSE) const;  
```  
  
### <a name="parameters"></a>매개 변수  
 `lprcBounds`  
 컨테이너 내에 있는 OLE 컨트롤의 범위에 대 한 포인터입니다. 클라이언트 영역이 아니라 하지만 경계와 스크롤 막대가 포함 하는 전체 컨트롤 영역.  
  
 `pPoint`  
 부모 (컨테이너)에 대 한 포인터를 컨트롤의 클라이언트 영역 좌표로 변환 해야 할 가리킵니다.  
  
 `bHitTest`  
 여부는 지점에서 수행 해야 하는 적중 테스트를 지정 합니다.  
  
### <a name="return-value"></a>반환 값  
 경우 `bHitTest` 는 **FALSE**, 반환 **HTNOWHERE**합니다. 경우 `bHitTest` 는 **TRUE**OLE 컨트롤의 클라이언트 영역에 착륙 부모 (컨테이너)를 가리키고 위치를 반환 하 고 적중 테스트 값을 다음 마우스 중 하나입니다.  
  
- **HTBORDER** 크기 조정 테두리 없는 창 테두리에 있습니다.  
  
- **HTBOTTOM** 창의 아래쪽 가로 테두리에 있습니다.  
  
- **HTBOTTOMLEFT** 창 테두리의 왼쪽 아래 모서리에 있습니다.  
  
- **HTBOTTOMRIGHT** 창 테두리의 오른쪽 아래 모서리에 있습니다.  
  
- **HTCAPTION** 제목 표시줄 영역에 있습니다.  
  
- **HTCLIENT** 클라이언트 영역에 있습니다.  
  
- **HTERROR** 화면 배경 또는 창 간의 구분선 (동일 **HTNOWHERE** 점을 제외 하 고는 **DefWndProc** 오류를 나타내는 데 시스템 경고음을 생성 하는 Windows 함수).  
  
- **HTGROWBOX** 크기 상자에서입니다.  
  
- **HTHSCROLL** 가로 스크롤 막대에 있습니다.  
  
- **HTLEFT** 창의 왼쪽된 테두리에 있습니다.  
  
- **HTMAXBUTTON** 에 최대화 단추입니다.  
  
- **HTMENU** 메뉴 영역에 있습니다.  
  
- **HTMINBUTTON** 단추는 최소화 합니다.  
  
- **HTNOWHERE** 화면 배경 또는 창 사이 경계선을 긋습니다.  
  
- **HTREDUCE** 단추는 최소화 합니다.  
  
- **HTRIGHT** 창의 오른쪽 테두리에 있습니다.  
  
- **HTSIZE** 크기 상자에서 (동일 **HTGROWBOX**).  
  
- **HTSYSMENU** 자식 창에 닫기 단추 또는 컨트롤 메뉴에 있습니다.  
  
- **HTTOP** 창의 상단 가로 테두리에 있습니다.  
  
- **HTTOPLEFT** 창 테두리의 왼쪽 위 모퉁이에 있습니다.  
  
- **HTTOPRIGHT** 창 테두리의 오른쪽 위 모서리에 있습니다.  
  
- **HTTRANSPARENT** 다른 창에서 다루는 현재 창에 있습니다.  
  
- **HTVSCROLL** 세로 스크롤 막대에 있습니다.  
  
- **HTZOOM** 에 최대화 단추입니다.  
  
### <a name="remarks"></a>주의  
 입력에 `pPoint` 은 부모 (컨테이너의 왼쪽된 위 모퉁이)의 원점을 기준으로 합니다. 출력에서 `pPoint` (컨트롤의 클라이언트 영역의 왼쪽된 위 모퉁이)에 OLE 컨트롤의 클라이언트 영역의 원점을 기준으로 합니다.

 Note 스톡 캡션 및 텍스트 속성 모두에 매핑되는 동일한 값입니다. 즉, 두 속성 중 하나를 변경한 속성을 모두 자동으로 변경 됩니다. 일반적으로 컨트롤에 스톡 캡션 또는 Text 속성 중 하나만 지원 해야 합니다.  
  
##  <a name="throwerror"></a>COleControl::ThrowError  
 신호를 사용자가 컨트롤에 오류가 발생 합니다.  
  
```  
void ThrowError(
    SCODE sc,  
    UINT nDescriptionID,  
    UINT nHelpID = -1);

 
void ThrowError(
    SCODE sc,  
    LPCTSTR pszDescription = NULL,  
    UINT nHelpID = 0);
```  
  
### <a name="parameters"></a>매개 변수  
 `sc`  
 상태 코드를 보고 하도록 값입니다. 가능한 코드 목록은 전체 문서를 참조 하십시오. [ActiveX 컨트롤: 고급 항목](../../mfc/mfc-activex-controls-advanced-topics.md)합니다.  
  
 `nDescriptionID`  
 문자열의 리소스 ID는 예외를 보고 하도록 합니다.  
  
 `nHelpID`  
 에 보고 하는 항목의 도움말 ID입니다.  
  
 `pszDescription`  
 보고 되는 예외에 대 한 설명을 포함 하는 문자열입니다.  
  
### <a name="remarks"></a>주의  
 이 함수만 될 내에서 호출 해야 Get 또는 Set 함수는 OLE 속성 또는 OLE 자동화 메서드의 구현입니다. 다른 시간에 발생 하는 오류를 알리기 위해 해야 할 경우에 주식 오류 이벤트를 발생 해야 합니다.  
  
##  <a name="transformcoords"></a>COleControl::TransformCoords  
 사이의 값을 조정 하는 변환 **HIMETRIC** 단위와 해당 컨테이너의 기본 단위입니다.  
  
```  
void TransformCoords(
    POINTL* lpptlHimetric,  
    POINTF* lpptfContainer,  
    DWORD flags);
```  
  
### <a name="parameters"></a>매개 변수  
 *lpptlHimetric*  
 에 대 한 포인터는 **POINTL** 좌표에 포함 된 구조 **HIMETRIC** 단위입니다.  
  
 *lpptfContainer*  
 에 대 한 포인터는 **POINTF** 좌표는 컨테이너의 단위 크기에 포함 된 구조체입니다.  
  
 `flags`  
 다음 값의 조합입니다.  
  
- **XFORMCOORDS_POSITION** 컨테이너의 위치입니다.  
  
- **XFORMCOORDS_SIZE** 컨테이너의 크기입니다.  
  
- **XFORMCOORDS_HIMETRICTOCONTAINER** 변환 **HIMETRIC** 단위 컨테이너의 단위입니다.  
  
- **XFORMCOORDS_CONTAINERTOHIMETRIC** 컨테이너의 단위를 변환 **HIMETRIC** 단위입니다.  
  
### <a name="remarks"></a>주의  
 처음 두 플래그 **XFORMCOORDS_POSITION** 및 **XFORMCOORDS_SIZE**, 좌표 위치 또는 크기를 처리할지 여부를 나타냅니다. 나머지 두 플래그에는 변환의 방향을 나타냅니다.  
  
##  <a name="translatecolor"></a>COleControl::TranslateColor  
 색상 값을 변환는 **OLE_COLOR** 데이터 형식으로 [COLORREF](http://msdn.microsoft.com/library/windows/desktop/dd183449) 데이터 형식입니다.  
  
```  
COLORREF TranslateColor(
    OLE_COLOR clrColor,  
    HPALETTE hpal = NULL);
```  
  
### <a name="parameters"></a>매개 변수  
 `clrColor`  
 A **OLE_COLOR** 데이터 형식입니다. 자세한 내용은 Windows를 참조 하십시오. [OleTranslateColor](http://msdn.microsoft.com/library/windows/desktop/ms694353) 함수입니다.  
  
 `hpal`  
 선택적 팔레트;에 대 한 핸들 수 **NULL**합니다.  
  
### <a name="return-value"></a>반환 값  
 솔리드를 정의 하는 RGB (빨강, 녹색, 파란색) 32 비트 색상 값에 가장 가까운 색은 `clrColor` 장치를 나타낼 수 있는 값입니다.  
  
### <a name="remarks"></a>주의  
 이 함수는 주식 ForeColor 및 BackColor 속성을 변환 하는 데 유용 **COLORREF** 사용 되는 형식을 [CDC](../../mfc/reference/cdc-class.md) 멤버 함수입니다.  
  
##  <a name="willambientsbevalidduringload"></a>COleControl::WillAmbientsBeValidDuringLoad  
 사용할지 여부를 결정 컨트롤 앰비언트 속성의 값은 기본값으로 영구 상태에서 이후에 로드 되는 경우.  
  
```  
BOOL WillAmbientsBeValidDuringLoad();
```  
  
### <a name="return-value"></a>반환 값  
 0이 아니면 나타냅니다 앰비언트 속성 올바르지 않습니다. 그렇지 않으면 앰비언트 속성 유효 해 집니다.  
  
### <a name="remarks"></a>주의  
 일부 컨테이너에 컨트롤 액세스 권한이 없을 수 앰비언트 속성의 재정의를 초기 호출 하는 동안 `COleControl::DoPropExchange`합니다. 이 경우에 컨테이너를 호출 하는 경우 [IPersistStreamInit::Load](http://msdn.microsoft.com/library/windows/desktop/ms680730) 또는 [IPersistStorage::Load](http://msdn.microsoft.com/library/windows/desktop/ms680557) 호출 하기 전에 [IOleObject::SetClientSite](http://msdn.microsoft.com/library/windows/desktop/ms684013) (즉, 준수 하지 않는 경우는 **OLEMISC_SETCLIENTSITEFIRST** 상태 비트)입니다.

# 명령줄 오류 D8045
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

\/clr 옵션으로 C 파일 'file'을\(를\) 컴파일할 수 없습니다.  
  
 **\/clr**를 사용하는 컴파일에는 C\+\+ 소스 코드 파일만 전달할 수 있습니다.  .c 파일을 .cpp 파일로 컴파일하려면 **\/TP**를 사용해야 합니다. 자세한 내용은 [\/Tc, \/Tp, \/TC, \/TP\(소스 파일 형식 지정\)](../../build/reference/tc-tp-tc-tp-specify-source-file-type.md)를 참조하십시오.  
  
 자세한 내용은 [\/clr\(공용 언어 런타임 컴파일\)](../../build/reference/clr-common-language-runtime-compilation.md)을 참조하십시오.  
  
 Visual C\+\+를 사용하여 ATL 응용 프로그램을 컴파일하는 경우에도 D8045 오류가 발생할 수 있습니다.  자세한 내용은 [방법: \/clr로 마이그레이션](../../dotnet/how-to-migrate-to-clr.md)를 참조하십시오.
